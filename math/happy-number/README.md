# Happy Number

**Источник:** LeetCode #202

## Условие

Дано положительное целое число `n`. На каждом шаге заменяем число суммой квадратов его цифр и повторяем процесс, пока число не станет равным `1` (тогда оно остаётся `1`), либо процесс зацикливается на числах, не включающих `1`. Вернуть `true`, если `n` — счастливое число (процесс заканчивается на `1`), иначе `false`.

## Ограничения

- `1 <= n <= 2^31 - 1`

## Мой подход

Рекурсия с отслеживанием уже встреченных сумм через `Set`, чтобы поймать зацикливание:

- На каждом шаге перевожу число в строку, разбиваю на цифры и считаю сумму их квадратов.
- Если сумма равна `1` — число счастливое, `true`.
- Если такая сумма уже встречалась раньше (`seen.has(sum)`) — значит процесс зациклился без единицы, `false`.
- Иначе запоминаю сумму в `seen` и рекурсивно повторяю процесс для неё.

Известно, что цепочка либо доходит до `1`, либо попадает в один и тот же короткий цикл (длиной около 8 чисел) — поэтому глубина рекурсии небольшая и не переполняет стек даже для `n` до `2^31 - 1`.

## Сложность

- Время: O(log n) на один шаг (количество цифр числа), количество шагов до завершения ограничено известным циклом
- Память: O(1) — набор промежуточных сумм в `seen` ограничен
