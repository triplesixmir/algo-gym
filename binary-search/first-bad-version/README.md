# First Bad Version

**Источник:** LeetCode #278

## Условие

Есть `n` версий `[1, 2, ..., n]`, все версии после первой плохой тоже плохие. Дан API `isBadVersion(version)`, возвращающий, плохая ли версия. Нужно найти первую плохую версию, минимизировав число обращений к API.

## Ограничения

- `1 <= bad <= n <= 2^31 - 1`

## Мой подход

Бинарный поиск "найти первое true" — сужаю диапазон, пока `left` не станет равен `right`:

- Если `isBadVersion(mid)` — `true`, первая плохая версия либо `mid`, либо ещё левее: `right = mid` (не `mid - 1`, потому что `mid` сам может быть ответом, его нельзя исключать из диапазона поиска).
- Если `false` — первая плохая версия точно правее: `left = mid + 1`.
- Когда `left === right` — цикл заканчивается, это и есть первая плохая версия.

Решение оформлено как фабрика (`function(isBadVersion) { return function(n) {...} }`), а не обычная функция — так задано условием LeetCode: `isBadVersion` внедряется как зависимость снаружи, а не является глобальной функцией.

## Сложность

- Время: O(log n) — бинарный поиск, минимизирует число вызовов `isBadVersion`
- Память: O(1)
