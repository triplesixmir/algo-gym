# Search Insert Position

**Источник:** LeetCode #35

## Условие

Дан отсортированный массив различных целых чисел `nums` и целевое значение `target`. Вернуть индекс `target`, если он найден в массиве. Если нет — вернуть индекс, на который его нужно вставить, чтобы массив остался отсортированным. Требуется алгоритм со сложностью O(log n).

## Ограничения

- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` содержит различные значения, отсортированные по возрастанию
- `-10^4 <= target <= 10^4`

## Мой подход

Бинарный поиск с двумя указателями `left` и `right`:

- На каждом шаге беру середину `mid` текущей области поиска.
- Если `nums[mid] === target` — нашли, возвращаю `mid`.
- Если `nums[mid] < target` — искомое значение правее, сдвигаю `left = mid + 1`.
- Если `nums[mid] > target` — искомое значение левее, сдвигаю `right = mid - 1`.
- Если цикл закончился (`left > right`) и совпадения не было — `target` в массиве нет. В этот момент `left` как раз указывает на первую позицию, где элемент не меньше `target`, то есть на нужное место для вставки.

## Сложность

- Время: O(log n) — на каждом шаге область поиска делится пополам
- Память: O(1) — используются только индексы, без дополнительных структур
