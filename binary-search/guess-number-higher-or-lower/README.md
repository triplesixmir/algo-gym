# Guess Number Higher Or Lower

**Источник:** LeetCode #374

## Условие

Загадано число от `1` до `n`. Нужно угадать его, используя предопределённое API `guess(num)`, которое возвращает:

- `-1` — если `num` больше загаданного числа;
- `1` — если `num` меньше загаданного числа;
- `0` — если угадано верно.

Вернуть загаданное число.

## Ограничения

- `1 <= n <= 2^31 - 1`
- `1 <= pick <= n`

## Мой подход

Бинарный поиск поверх внешнего API `guess`:

- Беру середину текущего диапазона `mid` и спрашиваю у `guess(mid)`.
- Если ответ `-1` — моя догадка выше загаданного числа, значит оно левее: `right = mid - 1`.
- Если ответ `1` — догадка ниже, число правее: `left = mid + 1`.
- Если ответ `0` — угадал, возвращаю `mid`.

Несмотря на то что `n` может доходить до `2^31 - 1` (что в языках с 32-битными int могло бы вызвать переполнение при вычислении середины), в JS числа хранятся как 64-битные double и безопасно работают с целыми вплоть до `2^53`, так что `Math.floor((left + right) / 2)` здесь безопасен без риска переполнения.

## Сложность

- Время: O(log n) — бинарный поиск
- Память: O(1)
