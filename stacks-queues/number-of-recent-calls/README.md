# Number Of Recent Calls

**Источник:** LeetCode #933

## Условие

Реализовать класс `RecentCounter`, который считает количество недавних запросов в пределах определённого временного окна:

- `RecentCounter()` — инициализирует счётчик без запросов.
- `ping(t)` — добавляет новый запрос на момент времени `t` (в миллисекундах) и возвращает количество запросов, произошедших за последние `3000` мс, включая текущий, то есть попадающих в диапазон `[t - 3000, t]`.

Гарантируется, что каждый вызов `ping` использует строго большее `t`, чем предыдущий.

## Ограничения

- `1 <= t <= 10^9`
- Каждый тест вызывает `ping` со строго возрастающими значениями `t`
- Не более `10^4` вызовов `ping`

## Мой подход

Скользящее окно поверх массива, без удаления устаревших элементов — вместо этого используется указатель `head`:

- В конструкторе — `requests` (история всех запросов) и `head` (индекс первого ещё актуального запроса), оба стартуют "пустыми".
- В `ping(t)`: добавляю `t` в конец `requests`, затем сдвигаю `head` вперёд, пока элемент под этим индексом устарел (`< t - 3000`).
- Раз `t` в каждом следующем вызове строго больше предыдущего (по условию), `head` никогда не двигается назад — значит суммарно за все вызовы он продвинется максимум на `n` шагов, а не будет пересчитываться заново на каждом `ping`. Это даёт амортизированную O(1) сложность на вызов, а не O(k) (где k — размер окна), как было бы при `shift()`/`filter()` на каждом пинге.
- Ответ — `requests.length - head`, то есть количество ещё актуальных записей.

## Сложность

- Время: O(1) амортизированно на вызов `ping` (в сумме за `n` вызовов — O(n))
- Память: O(n) — массив `requests` растёт с каждым вызовом (устаревшие элементы физически не удаляются, только пропускаются через `head`)
