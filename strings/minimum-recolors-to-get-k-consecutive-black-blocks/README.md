# Minimum Recolors To Get K Consecutive Black Blocks

**Источник:** LeetCode #2379

## Условие

Дана строка `blocks` длины `n`, где каждый символ — `'W'` (белый) или `'B'` (чёрный) блок, и число `k` — желаемое количество подряд идущих чёрных блоков. За одну операцию можно перекрасить белый блок в чёрный. Вернуть минимальное количество операций, необходимое для получения хотя бы одного участка из `k` подряд идущих чёрных блоков.

## Ограничения

- `n == blocks.length`
- `1 <= n <= 100`
- `blocks[i]` — `'W'` или `'B'`
- `1 <= k <= n`

## Мой подход

Скользящее окно фиксированного размера `k` — аналогично Maximum Average Subarray I, только вместо суммы считаю количество белых блоков:

- `w` — количество `'W'` в текущем окне длины `k`. Стартовое окно (первые `k` символов) считаю явно, до основного цикла.
- Дальше окно скользит по одному символу: прибавляю новый символ справа, вычитаю тот, что "выпал" слева (`blocks[i - k]`).
- На каждом шаге обновляю `ans` — минимальное количество белых блоков среди всех окон длины `k`.

Минимум белых блоков в любом окне размера `k` — это и есть минимальное число перекрасок: чтобы получить `k` чёрных подряд именно в этом окне, нужно перекрасить ровно столько белых блоков, сколько их там есть.

## Сложность

- Время: O(n) — один проход по строке
- Память: O(1)
