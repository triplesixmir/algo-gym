# Isomorphic Strings

**Источник:** LeetCode #205

## Условие

Даны строки `s` и `t`. Определить, изоморфны ли они: все вхождения символа в `s` можно заменить другим символом, чтобы получить `t`, с сохранением порядка. Два разных символа не могут отображаться в один и тот же символ, но символ может отображаться сам в себя.

## Ограничения

- `1 <= s.length <= 5 * 10^4`
- `t.length == s.length`
- `s` и `t` состоят из любых валидных ASCII-символов

## Мой подход

Вместо явных двух отображений символ→символ сравниваю индексы **первого вхождения** каждого символа:

- `charIndexS[c]` и `charIndexT[c]` хранят индекс, на котором символ `c` встретился впервые в `s` и `t` соответственно (записываются только один раз, при первом появлении).
- На каждой позиции `i` сравниваю `charIndexS[s[i]]` с `charIndexT[t[i]]`. Если строки изоморфны, то при любом повторном появлении символа `s[i]` соответствующий ему `t[i]` должен был впервые встретиться ровно в той же позиции — иначе либо один символ пытается отобразиться в два разных (нарушение функции), либо два разных символа отображаются в один и тот же (нарушение биективности).
- Если хоть раз индексы не совпали — строки не изоморфны.

## Сложность

- Время: O(n) — один проход по строкам
- Память: O(k), где k — число уникальных символов (не более 128 для ASCII)
