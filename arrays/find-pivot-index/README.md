# Find Pivot Index

**Источник:** LeetCode #724

## Условие

Дан массив целых чисел `nums`. Найти pivot index — индекс, для которого сумма всех чисел строго левее равна сумме всех чисел строго правее. Если индекс находится на краю массива, соответствующая сумма считается равной `0`. Вернуть самый левый такой индекс, если он есть, иначе `-1`.

## Ограничения

- `1 <= nums.length <= 10^4`
- `-1000 <= nums[i] <= 1000`

## Мой подход

Считаю общую сумму массива `total` один раз, а дальше на каждом шаге получаю правую сумму без пересчёта:

- `rightTotal = total - leftTotal - nums[i]` — это сумма всех элементов, кроме уже учтённых слева и текущего элемента.
- Сравниваю `leftTotal` и `rightTotal`. Если совпали — это и есть искомый pivot index, причём самый левый, потому что иду по массиву слева направо и возвращаю сразу при первом совпадении.
- Если не совпало — добавляю `nums[i]` к `leftTotal` и перехожу к следующей позиции.
- Если ни на одном индексе не нашлось совпадения — возвращаю `-1`.

## Сложность

- Время: O(n) — один проход на подсчёт `total` (через `reduce`) плюс один основной проход
- Память: O(1)
