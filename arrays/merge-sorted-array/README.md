# Merge Sorted Array

**Источник:** LeetCode #88

## Условие

Даны два отсортированных по неубыванию массива `nums1` и `nums2`, а также числа `m` и `n` — количество значащих элементов в каждом из них. Нужно слить их в один отсортированный массив, сохранив результат внутри `nums1`. Длина `nums1` равна `m + n`: первые `m` элементов — значащие, последние `n` — заглушки (нули), которые нужно перезаписать.

## Ограничения

- `nums1.length == m + n`
- `nums2.length == n`
- `0 <= m, n <= 200`
- `1 <= m + n <= 200`
- `-10^9 <= nums1[i], nums2[j] <= 10^9`

Follow-up: решить за O(m + n).

## Мой подход

Слияние с конца массива с помощью трёх указателей, чтобы не перезаписывать ещё не обработанные элементы `nums1`:

- `p1` — указатель на последний значащий элемент `nums1`, `p2` — на последний элемент `nums2`, `k` — позиция записи (последний индекс итогового массива).
- Пока оба указателя валидны, сравниваю `nums1[p1]` и `nums2[p2]`, больший элемент ставлю в `nums1[k]`, сдвигаю соответствующий указатель и `k` — на каждом шаге.
- Если `nums2` заканчивается первым — цикл просто завершается, оставшиеся элементы `nums1` уже стоят на своих местах, трогать их не нужно.
- Если `nums1` заканчивается первым — доливаю оставшиеся элементы `nums2` вторым циклом.

Запись всегда идёт в ещё не тронутую (заполненную нулями) часть массива, поэтому перезаписи нужных данных не происходит.

## Сложность

- Время: O(m + n) — каждый элемент обоих массивов обрабатывается один раз
- Память: O(1) — слияние на месте, без дополнительных структур
