# Kids With The Greatest Number Of Candies

**Источник:** LeetCode #1431

## Условие

Есть `n` детей с конфетами. Дан массив `candies`, где `candies[i]` — количество конфет у i-го ребёнка, и число `extraCandies` — количество дополнительных конфет. Вернуть булев массив `result`, где `result[i] = true`, если, отдав все `extraCandies` i-му ребёнку, он будет иметь наибольшее количество конфет среди всех детей (возможно, наравне с другими).

## Ограничения

- `n == candies.length`
- `2 <= n <= 100`
- `1 <= candies[i] <= 100`
- `1 <= extraCandies <= 50`

## Мой подход

- Сначала нахожу `max` — максимум по массиву `candies`, один раз до основного прохода.
- Затем `map` по каждому ребёнку: если `candies[i] + extraCandies >= max`, значит с добавкой у него будет не меньше конфет, чем у текущего лидера — `true`, иначе `false`.

Важно посчитать `max` заранее, а не внутри `map` — иначе на каждой итерации заново пересчитывался бы максимум по всему массиву, что дало бы O(n²) вместо O(n).

## Сложность

- Время: O(n) — один проход на поиск максимума, один проход на map
- Память: O(n) — под результирующий массив
