# Intersection Of Two Arrays II

**Источник:** LeetCode #350

## Условие

Даны два массива целых чисел `nums1` и `nums2`. Вернуть массив их пересечения, где каждый элемент должен появиться столько раз, сколько он встречается в обоих массивах (минимум из двух количеств). Порядок не важен.

## Ограничения

- `1 <= nums1.length, nums2.length <= 1000`
- `0 <= nums1[i], nums2[i] <= 1000`

## Мой подход

Два указателя после сортировки — в отличие от [Intersection of Two Arrays](../intersection-of-two-arrays), здесь важно количество повторов, а не просто факт наличия элемента, поэтому `Set` не подходит.

- Сортирую оба массива, чтобы можно было идти по ним двумя указателями синхронно.
- Если `nums1[i] < nums2[k]` — текущий элемент `nums1` точно не найдёт пару справа (там всё больше), двигаю `i`.
- Если `nums1[i] > nums2[k]` — аналогично, двигаю `k`.
- Если равны — это совпадение, добавляю в результат и двигаю **оба** указателя (иначе один и тот же элемент может учесться повторно).

## Сложность

- Время: O((n + m) log(n + m)) — доминирует сортировка обоих массивов
- Память: O(1) дополнительной памяти (не считая результат); сортировка мутирует исходные массивы на месте
