# Intersection Of Two Arrays

**Источник:** LeetCode #349

## Условие

Даны два массива целых чисел `nums1` и `nums2`. Вернуть массив их пересечения. Каждый элемент результата должен быть уникальным, порядок не важен.

## Ограничения

- `1 <= nums1.length, nums2.length <= 1000`
- `0 <= nums1[i], nums2[i] <= 1000`

## Мой подход

Перевожу оба массива в `Set` (сразу убирает дубликаты и даёт проверку на вхождение за O(1)), затем перебираю элементы одного множества и проверяю, есть ли они во втором:

- `s1`, `s2` — множества уникальных значений из `nums1` и `nums2`.
- Прохожу по `s1` и для каждого элемента проверяю `s2.has(e)` — если есть в обоих, добавляю в результат.

**Альтернатива:** есть более короткий способ через новый (2024) метод `Set.prototype.intersection()`:

```js
const set1 = new Set(nums1);
const set2 = new Set(nums2);
return Array.from(set1.intersection(set2));
```

Он даёт тот же результат за меньшее количество кода, но не показывает, как пересечение множеств работает "под капотом" — это просто вызов готового API. Для тренировки алгоритмов основным выбран вариант с явным циклом, как более показательный.

## Сложность

- Время: O(n + m) — построение множеств и проход по меньшему из них
- Память: O(n + m) — под множества
