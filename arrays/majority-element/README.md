# Majority Element

**Источник:** LeetCode #169

## Условие

Дан массив `nums` размера `n`. Вернуть majority element — элемент, который встречается более `⌊n / 2⌋` раз. Гарантируется, что такой элемент в массиве существует.

Follow-up: решить за линейное время и O(1) памяти.

## Ограничения

- `n == nums.length`
- `1 <= n <= 5 * 10^4`
- `-10^9 <= nums[i] <= 10^9`

## Мой подход

Алгоритм голосования большинства Бойера-Мура (Boyer-Moore Voting Algorithm):

- `cand` — текущий кандидат в majority element, `count` — "перевес" голосов за него.
- Если `count === 0` — текущий кандидат исчерпан, беру новым кандидатом текущий элемент.
- Если текущий элемент совпадает с кандидатом — увеличиваю перевес (`count++`).
- Если не совпадает — уменьшаю (`count--`), как будто этот элемент "отменяет" один голос кандидата.

Идея работает потому, что majority element встречается больше `⌊n/2⌋` раз — то есть чаще, чем все остальные элементы вместе взятые. Каждый непохожий элемент может "погасить" не более одного голоса кандидата, но раз голосов за majority element в сумме больше, чем у всех остальных элементов, он гарантированно останется кандидатом к концу прохода.

## Сложность

- Время: O(n) — один проход по массиву
- Память: O(1) — без дополнительных структур (`Map`/`Set` для подсчёта частот не нужны)
