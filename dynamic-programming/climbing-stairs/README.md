# Climbing Stairs

**Источник:** LeetCode #70

## Условие

Нужно подняться по лестнице из `n` ступеней. За один раз можно подняться на 1 или на 2 ступени. Сколько существует различных способов подняться наверх?

## Ограничения

- `1 <= n <= 45`

## Мой подход

Динамическое программирование (снизу вверх), по сути — числа Фибоначчи:

- Последний шаг к ступеньке `n` — это либо шаг на 1 (пришли с `n-1`), либо шаг на 2 (пришли с `n-2`). Значит `climbStairs(n) = climbStairs(n-1) + climbStairs(n-2)`.
- Базовые случаи `n = 1, 2, 3` совпадают с самим `n` (1, 2, 3 способа соответственно) — обрабатываю их одной ранней проверкой.
- Дальше вместо хранения всего массива значений храню только два последних (`prev1`, `prev2`) и на каждом шаге считаю следующее значение — экономлю память.

## Сложность

- Время: O(n) — один проход от 3 до n
- Память: O(1) — хранятся только два предыдущих значения, а не весь массив
