# Island Perimeter

**Источник:** LeetCode #463

## Условие

Дана сетка `grid` размером `row x col`, где `grid[i][j] = 1` — суша, `grid[i][j] = 0` — вода. Клетки соединены по горизонтали/вертикали (не по диагонали). Сетка полностью окружена водой, остров ровно один и без "озёр" внутри. Найти периметр острова.

## Ограничения

- `row == grid.length`
- `col == grid[i].length`
- `1 <= row, col <= 100`
- `grid[i][j]` — `0` или `1`
- В сетке ровно один остров

## Мой подход

Один проход по всей сетке без обхода графа (DFS тут не обязателен, раз нужно просто посчитать периметр, а не обойти сам остров):

- Для каждой клетки земли изначально закладываю `4` — как будто все 4 стороны открыты (граничат с водой или с краем сетки).
- Проверяю **только** соседа сверху и соседа слева (а не все 4 направления): если там тоже земля, вычитаю `2` — по одной стороне у текущей клетки и у соседа, они больше не "открыты", раз граничат друг с другом.
- Проверка именно "сверху и слева" (не "снизу и справа") — не случайна: идя по сетке слева направо и сверху вниз, сосед сверху/слева уже гарантированно обработан на текущий момент, а сосед справа/снизу — ещё нет. Так каждая внутренняя граница между двумя клетками земли учитывается ровно один раз, а не дважды.

## Сложность

- Время: O(rows × cols) — один проход по всей сетке
- Память: O(1) — без дополнительных структур
