# Hamming Distance

**Источник:** LeetCode #461

## Условие

Расстояние Хэмминга между двумя целыми числами — количество позиций, в которых соответствующие биты различаются. Даны целые числа `x` и `y`, вернуть расстояние Хэмминга между ними.

## Ограничения

- `0 <= x, y <= 2^31 - 1`

## Мой подход

Побитовый XOR + подсчёт единичных битов:

- `x ^ y` — побитовый XOR над самими числами (не строками!). В результате единица стоит ровно в тех позициях, где биты `x` и `y` отличались, ноль — где совпадали. Это как раз даёт "карту различий".
- `.toString(2)` переводит **результат** XOR в двоичную строку.
- Считаю количество символов `'1'` в этой строке — это и есть расстояние Хэмминга.

Важный момент, на котором споткнулся по пути: нельзя сначала переводить `x` и `y` в двоичные строки, а потом применять `^` к строкам — JS в этом случае приведёт строки к числу через десятичную интерпретацию (`Number("1011101")` — это 1011101 в десятичной системе, а не 93 в двоичной), и результат будет мусором. XOR должен применяться напрямую к самим числам, а в двоичную строку переводится уже итог операции.

## Сложность

- Время: O(log(max(x, y))) — количество битов в числе
- Память: O(log(max(x, y))) — под строку и массив символов
